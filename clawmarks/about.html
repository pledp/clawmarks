clawmarks.

In Clawmarks, you’re an air traffic controller at your local airport, HEL. Manage flights, avoid collisions, and keep passengers safe!

The game centers around air traffic tasks. These tasks appear rapidly, and it’s the controller’s responsibility to handle them swiftly and efficiently.

tasks.

The game periodically adds more flights for the controller to handle, each flight is accompanied by a task. Some tasks, like the Fire task, only show up at random events.

You solve tasks by writing in the input field a suitable command for the task.

There are several types of tasks available:

  • Altitude

    Shows up randomly accompanying a flight.

    To solve the task:
    AY380 - HEL -> SOME - Set altitude to 5359,

    you write:
    AY380 altitude 5359

  • Takeoff

    Shows up randomly accompanying a flight.

    To solve the task:
    AY570 - HEL -> SOME - Permit takeoff,

    you write:
    AY570 permit takeoff HEL

  • Heading

    Shows up randomly accompanying a flight.

    To solve the task:
    AX230 - HEL -> SOME - Set heading to 075,

    you write:
    AX230 heading 075

  • Landing

    Shows up randomly accompanying a flight.

    To solve the task:
    AX645 - HEL -> SOME - Permit landing,

    you write:
    AX645 permit landing HEL

  • Fire

    Occurs randomly as an emergency situation at the airport.

    To solve the task:
    Fire at runway 3,

    you write:
    fire runway 3

server.

The Flask server offers a lightweight and flexible platform that efficiently facilitates the backend functions of the game. It allows the construction of a REST API that the game's frontend uses to communicate with the game database. This API handles all data transmission and reception between the players and the game server, making the game dynamic and interactive.

User Account Management:

Flask enables user registration and login. This is visible in the game as players can create user accounts, log in, and start the game from where they last stopped.

Real-time Data Processing:

Flask handles game events such as task completion and score calculation in real time. This appears in the game as players receive immediate feedback on task completion, making the game interactive and engaging.

Task Management:

Flask manages the creation and distribution of tasks to players. This feature is visible in the game as players continuously receive new tasks that help them earn points.

login & stuff

Our game's login system is designed with an emphasis on user privacy and security. It does not collect or store personal information, focusing instead on enhancing gameplay and user experience.

Technical Implementation:

The login process utilizes the Flask web framework, enabling simple yet effective user management. When a player creates a new account or logs in, the game server processes the data as follows:

Registration:

Players enter their chosen username and password. The Flask server checks if the username is already in use. If the username is available, the information is stored in the database in encrypted form.

Login:

Players enter their username and password. Flask verifies the information against the database. If the credentials match, players can continue the game from where they last left off.

Security and Privacy:

We do not store personal information such as email addresses or other identifying details. We employ strong password encryption to ensure user data remains protected. Login details are only stored as necessary for the operation of the game.

This system ensures the privacy of our players while providing a seamless and secure gaming experience. The login mechanism is designed to be quick and non-intrusive, allowing players to focus on what's important – enjoying the game without worrying about data security.
